# Contribución al proyecto Hermes

Este documento define las reglas y estándares para contribuir al proyecto Hermes.

## Estándares de código
- Python 3.10+
- PEP8 como guía base
- Arquitectura modular
- Tests obligatorios para cada módulo nuevo
- Documentación mínima por archivo

## Estructura de commits
- `feat:` nuevas funcionalidades
- `fix:` correcciones
- `docs:` documentación
- `refactor:` mejoras internas
- `test:` pruebas

## Flujo de trabajo
1. Crear rama nueva
2. Implementar cambios
3. Ejecutar tests
4. Crear PR
